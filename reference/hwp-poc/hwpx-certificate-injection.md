# HWPX 소독증명서 데이터 주입 워크플로우

## 목적

소독증명서 HWPX 템플릿 파일에 고객/방문 데이터를 프로그래밍으로 주입하여, 기존 양식(라벨, 테두리, 레이아웃)은 그대로 유지한 채 빈 칸에만 값을 채운 완성된 증명서를 생성한다.

---

## 1. 배경지식: HWP vs HWPX

### HWP (바이너리, `.hwp`)
- OLE 복합문서 포맷 (Microsoft Compound Document)
- 내부 스트림이 zlib으로 압축된 바이너리 레코드
- 텍스트를 수정하면 `PARA_HEADER`의 `nChars`, `PARA_CHAR_SHAPE`, `PARA_LINE_SEG` 등 연관 레코드를 모두 동기화해야 함
- **동기화 실패 시 "문서가 손상되었거나 변조되었을 가능성이 있습니다" 에러 발생**
- 결론: 프로그래밍으로 안전하게 편집하기 매우 어려움. 사용하지 않는다.

### HWPX (XML 기반, `.hwpx`)
- 실체는 ZIP 파일
- 내부에 XML로 문서 구조가 기술됨
- 텍스트는 `<hp:t>` 태그 안에 평문으로 존재
- XML만 수정하면 되므로 레코드 동기화 문제가 없음
- **결론: HWPX를 사용한다.**

### HWPX 내부 구조

```
소독증명서.hwpx (ZIP)
├── mimetype
├── version.xml
├── settings.xml
├── META-INF/
│   ├── container.xml
│   ├── container.rdf
│   └── manifest.xml
├── Contents/
│   ├── content.hpf          ← 문서 메타데이터
│   ├── header.xml           ← 글꼴, 스타일 등 정의
│   └── section0.xml         ← ★ 본문 내용 (여기만 수정)
└── Preview/
    ├── PrvText.txt
    └── PrvImage.png
```

## 2. 핵심 원칙: "치환이 아니라 삽입"

### 잘못된 접근 (치환)
```
원본:  "상호(명칭)"
결과:  "맛있는분식"        ← 라벨이 사라짐
```

### 올바른 접근 (삽입)
```
원본:  "상호" | "(명칭)" | "                      "
결과:  "상호" | "(명칭)" | "                      " | "맛있는분식"
                                                      ↑ 새 run 추가
```

기존 텍스트(라벨)는 절대 건드리지 않는다. 빈 칸(공백 문자열)에 값을 채우거나, 빈 칸 뒤에 새로운 `<hp:run>` 요소를 추가한다.

## 3. XML 구조 이해

### 계층 구조

```
<hp:tbl>                          ← 테이블
  <hp:tr>                         ← 행 (row)
    <hp:tc>                       ← 셀 (cell)
      <hp:cellAddr col="0" row="0"/>
      <hp:subList>                ← 셀 내용 컨테이너
        <hp:p>                    ← 문단 (paragraph)
          <hp:run charPrIDRef="6">   ← 텍스트 런 (글꼴/스타일 참조)
            <hp:t>상호</hp:t>         ← ★ 실제 텍스트
          </hp:run>
          <hp:run charPrIDRef="7">
            <hp:t>(명칭)</hp:t>
          </hp:run>
          <hp:linesegarray>...</hp:linesegarray>  ← 줄 배치 정보
        </hp:p>
        <hp:p>                    ← 두 번째 문단 (여기에 값이 들어감)
          <hp:run charPrIDRef="7">
            <hp:t>                      </hp:t>  ← 빈 칸
          </hp:run>
        </hp:p>
      </hp:subList>
    </hp:tc>
```

### 핵심 요소 설명

| 요소 | 역할 |
|------|------|
| `<hp:tc>` | 테이블 셀. `<hp:cellAddr>`로 위치 식별 |
| `<hp:subList>` | 셀 안의 콘텐츠 컨테이너 |
| `<hp:p>` | 문단. 하나의 셀에 여러 문단 가능 |
| `<hp:run>` | 텍스트 런. `charPrIDRef`로 글꼴/크기 스타일 참조 |
| `<hp:t>` | 실제 텍스트 내용 |
| `<hp:linesegarray>` | 줄 배치 메타데이터. run 삽입 시 이 앞에 넣어야 함 |
| `charPrIDRef` | `header.xml`에 정의된 글자 속성(CharShape) ID 참조 |

## 4. 데이터 주입 방식 3가지

### 방식 A: 기존 빈 칸 run의 텍스트에 값 이어붙이기

빈 칸 run이 하나뿐이고, 그 안에 값을 넣으면 되는 경우.

```python
# 템플릿: "                      "
# 결과:   "                      일반소독 (연무소독)"
set_run_text(runs[0], f'                      {value}')
```

**사용 셀**: 종류(cell[15]), 약품(cell[16])

### 방식 B: 빈 칸 run 뒤에 새 run 추가

라벨과 값이 시각적으로 분리되어야 하는 경우. 기존 빈 칸 run은 유지하고, 뒤에 새 run을 삽입한다.

```python
# 템플릿: run[0]="                      "
# 결과:   run[0]="                      " → run[1]="맛있는분식" (새로 추가)
append_run_to_para(para, value, charPrIDRef='6')
```

**사용 셀**: 상호(cell[3]), 소재지(cell[5]), 직위(cell[7]), 성명(cell[8])

### 방식 C: 기존 run의 텍스트 직접 수정

빈 칸이 구조적 의미를 가지는 경우 (예: `' 제       호'`의 빈 자리에 번호를 넣는 경우).

```python
# 템플릿: " 제       호"
# 결과:   " 제    7   호"
set_run_text(runs[0], f' 제    {cert_no}   호')
```

**사용 셀**: 증명서 번호(cell[1]), 소독기간(cell[12]), 날짜(cell[17].p[3]), 면적(cell[4])

## 6. 전체 처리 흐름

```
1. ZIP 읽기
   └─ zipfile.ZipFile(template.hwpx) → 모든 파일을 메모리에 로드

2. XML 파싱
   └─ Contents/section0.xml → ElementTree.fromstring()

3. 셀 목록 수집
   └─ root.iter('{hp_ns}tc') → cells 리스트

4. 각 셀에 데이터 주입
   └─ get_cell_paragraphs(cell) → paragraphs
   └─ paragraph 내 runs 탐색
   └─ 방식 A/B/C 중 적절한 방식으로 주입

5. XML 직렬화
   └─ ET.tostring() → UTF-8 바이트

6. ZIP 재생성
   └─ 수정된 section0.xml 포함하여 새 ZIP 생성
```

## 7. 주의사항 및 함정

### 7-1. namespace 등록 필수

ElementTree는 XML을 파싱할 때 namespace prefix를 잃어버린다. `ET.register_namespace()`로 모든 namespace를 사전 등록하지 않으면, 출력 XML에서 `ns0:`, `ns1:` 같은 자동 prefix가 붙어 한글 프로그램이 인식하지 못한다.

```python
ET.register_namespace('hp', 'http://www.hancom.co.kr/hwpml/2011/paragraph')
ET.register_namespace('hs', 'http://www.hancom.co.kr/hwpml/2011/section')
# ... 나머지 모든 namespace
```

### 7-2. run 삽입 위치: linesegarray 앞

`<hp:p>` 안에 `<hp:linesegarray>`가 있으면, 새 `<hp:run>`은 반드시 그 앞에 삽입해야 한다. 뒤에 넣으면 한글이 무시하거나 오류를 낸다.

```python
lineseg = p.find(f'{HP_NS}linesegarray')
if lineseg is not None:
    idx = list(p).index(lineseg)
    p.insert(idx, new_run)    # linesegarray 바로 앞
```

### 7-3. paragraph 인덱스 주의

하나의 셀(`<hp:tc>`)에 여러 `<hp:p>`가 있을 수 있다. 라벨 텍스트가 있는 paragraph와 빈 칸 paragraph의 인덱스를 정확히 구분해야 한다.

**실패 사례**: cell[1]에서 `paras[0]`을 수정했으나, 실제 텍스트는 `paras[1]`에 있었음. 결과적으로 빈 paragraph에 새 텍스트가 들어가고, 원본 텍스트도 그대로 남아 중복이 발생했다.

**규칙**: 반드시 셀 구조를 먼저 분석한 뒤, 텍스트가 있는 정확한 paragraph/run 인덱스를 확인한다.

### 7-4. charPrIDRef 값

새 run을 추가할 때 `charPrIDRef`는 `header.xml`에 정의된 글자 스타일 ID를 참조한다. 잘못된 ID를 넣으면 글꼴/크기가 의도와 다르게 표시된다. 예시 파일에서 사용된 ID를 참고하여 적절한 값을 사용한다.

### 7-5. XML declaration

`ET.tostring()`은 xml declaration을 포함하지 않으므로, 수동으로 추가해야 한다:

```python
xml_declaration = '<?xml version="1.0" encoding="UTF-8" standalone="yes" ?>'
output = (xml_declaration + ET.tostring(root, encoding='unicode')).encode('utf-8')
```

## 8. 검증 방법

생성된 HWPX 파일은 반드시 한글 프로그램에서 열어서 다음을 확인한다:

1. 파일이 에러 없이 열리는가
2. 기존 라벨(상호, 소재지, 종류 등)이 그대로 있는가
3. 주입된 데이터가 올바른 셀의 올바른 위치에 표시되는가
4. 테이블 테두리, 레이아웃이 깨지지 않았는가
5. 글꼴과 크기가 자연스러운가

## 10. 입력 데이터 명세

### 데이터 출처별 분류

| 필드 | 출처 | 설명 |
|------|------|------|
| 증명서 번호 (제 ○ 호) | **시스템 자동채번** | 아래 "발급번호 채번 규칙" 참조 |
| 상호(명칭) | 고객 DB | 소독 대상 시설명 |
| 실시 면적 / 용적 | 고객 DB | ㎡, ㎥ |
| 소재지 | 고객 DB | 대상 시설 주소 |
| 직위 / 성명 | 고객 DB | 시설 관리(운영)자 |
| 소독기간 | 방문 기록 | 소독 실시일 (시작~종료) |
| 종류 | 방문 기록 | 소독 방법 (일반소독, 연무소독 등) |
| 약품 사용 내용 | 방문 기록 | 약제명, 희석배율, 사용량 |
| 날짜 (년/월/일) | 방문 기록 | 증명서 발급일 |
| 소독 실시자 상호(명칭) | 업체(Tenant) 설정 | 소독업체명 |
| 소독 실시자 소재지 | 업체(Tenant) 설정 | 소독업체 주소 |
| 성명(대표자) | 업체(Tenant) 설정 | 소독업체 대표자명 |

### 발급번호 채번 규칙

"제 ○ 호"는 소독증명서의 일련번호(발급번호)이다.

**설계 배경**: 기존에 오프라인으로 사업을 운영하던 업체는 이미 발급한 증명서가 있다. 시스템이 1부터 강제로 시작하면 기존 발급 이력과 충돌한다.

**규칙**:
- 최초 설정 시 사용자가 "다음 발급번호 시작값"을 직접 입력 (예: 기존에 150호까지 발급했으면 151)
- 이후 증명서 발급 시 시스템이 자동으로 +1 증가
- 수동 번호 수정 옵션도 제공 (번호 건너뛰기 등 대응)

**DB 설계**:
- `Tenant(업체)` 설정에 `next_serial_number` 필드
- `Certificate` 테이블에 `serial_number` 컬럼
