# Phase 3: 스케줄 관리 + 캘린더

## 목표
캘린더 뷰에서 방문 예정 건 확인, 스케줄 CRUD, 방문 건(visits) 자동 생성

## 선행 조건
- Phase 2 완료 (고객 관리, 스케줄 기본 생성)

## 작업 목록

### 3-1. 캘린더 뷰
- [ ] 월간 캘린더 뷰 — 방문 건을 날짜별로 표시
- [ ] 주간 캘린더 뷰
- [ ] 상태별 색상 구분 (scheduled: 파란색, completed: 초록색, missed: 빨간색)
- [ ] 캘린더 날짜 클릭 → 해당 일 방문 건 목록 표시
- [ ] 방문 건 클릭 → 방문 상세 페이지 이동

### 3-2. 방문 건(Visits) 자동 생성
- [ ] 스케줄 기반 visits 자동 생성 로직 (주기에 따라 미래 방문 건 생성)
- [ ] 하절기/동절기 전환 시 주기 자동 변경 로직
- [ ] 방문 완료 시 다음 방문 예정일 자동 갱신 + 새 visit 생성

### 3-3. 미완료 건 처리
- [ ] 예정일 경과 시 status를 `missed`로 자동 변경 (API Route 또는 앱 로딩 시 체크)
- [ ] 캘린더에서 missed 건 빨간색 표시
- [ ] missed 건 재방문 스케줄 수동 등록 기능

### 3-4. 오늘 할 일 목록
- [ ] 오늘 예정된 방문 건 목록 API
- [ ] 기사별 오늘 할 일 필터

### 3-5. 알림 (인앱)
- [ ] 방문 하루 전 인앱 알림 생성
- [ ] 방문 당일 인앱 알림 생성
- [ ] 알림 목록 페이지
- [ ] 알림 읽음 처리

## 산출물
- 월간/주간 캘린더 뷰
- 방문 건 자동 생성/갱신
- 미완료 건 자동 감지
- 인앱 알림
