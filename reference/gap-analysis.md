# 방역매니저 GAP 분석

> 작성일: 2026-04-21
> 기준: 현재 코드베이스 vs 한국 방역 SaaS 이상형

한국 방역 시장 특성과 GorillaDesk 등 미국 SaaS를 벤치마킹해 도출한 "이상적인 한국 방역 SaaS" 대비 현재 방역매니저의 부족한 기능을 정리한 문서.

---

## 현재 방역매니저가 잘 갖춘 것

| 기능 | 완성도 | 비고 |
|------|--------|------|
| 고객/시설 관리 | Full | 14개 시설 유형, 법정 주기 데이터 내장 |
| 방문 일정 관리 | Full | 캘린더 + 목록 + 자동 상태 변경 |
| 소독증명서 발급 | Full | HWPX + PDF 자동 생성 |
| 직원 관리 + RBAC | Full | admin/member 권한 분리 |
| 대시보드 | Full | 오늘/주간/월간 요약 |
| 멀티테넌트 | Full | tenant_id 기반 완전 격리 |

---

## P0 — 한국 시장 킬러 피처 (경쟁 차별화)

### 1. 연간 의무소독 현황 추적

**현재 상태**
- `getCycleMonths()` 함수로 시설별 소독 주기 데이터는 존재
- 하지만 "고객 A는 올해 9회 중 4회 완료" 같은 추적 뷰가 없음

**필요한 것**
- 고객별 연간 계획 vs 실적 대시보드
- 시설 유형 기반 법정 의무 횟수 자동 계산
- 미이행 경고 알림 (과태료 리스크 방지)

**임팩트**
- 미국 SaaS에 없는 한국 고유 킬러 피처
- 감염병예방법 의무 이행 증명 도구로 활용 가능

---

### 2. "이번 달 방문해야 할 고객" 자동 추천

**현재 상태**
- 캘린더는 이미 등록된 방문만 표시
- 정기 계약 기반으로 이번 달 가야 하는데 아직 일정이 안 잡힌 고객을 알려주지 않음

**필요한 것**
- 정기 계약 + 법정 주기 기반으로 "이번 달 방문 대상 고객" 자동 추출
- 대시보드에 "미배정 고객" 위젯

**임팩트**
- 이게 없으면 결국 엑셀로 돌아감
- SaaS 핵심 가치(누락 방지)의 근간

---

### 3. 정기소독 계약 관리

**현재 상태**
- `clients` 테이블에 계약 정보가 없음 (시작일, 종료일, 금액, 소독 횟수)
- 방문을 수동으로 하나씩 등록해야 함

**필요한 것**
- `contracts` 테이블 추가 (1년 단위 계약이 한국 표준)
- 계약 등록 → 연간 방문 일정 자동 생성
- 계약 만료 30~60일 전 갱신 알림

**제안 데이터 모델**
```sql
contracts
├── client_id              -- 고객/시설
├── contract_number        -- 계약번호
├── start_date, end_date   -- 1년 기간
├── auto_renew             -- 자동 갱신 여부
├── renewal_notice_days    -- 만료 N일 전 알림
├── annual_service_count   -- 연간 횟수
├── service_months (jsonb) -- 실시 월
├── billing_type           -- per_visit / monthly / quarterly / annual
├── unit_price / monthly_price
├── total_amount           -- 연간 총액
├── payment_terms          -- prepaid / postpaid
├── status                 -- draft / active / expired / cancelled / renewed
└── original_contract_id   -- 갱신 시 이전 계약 참조
```

**임팩트**
- 한국 방역 업계 실제 업무 흐름(1년 계약 기반)과 일치
- 모든 P0/P1 기능의 기반 데이터

## P1 — 운영 효율화

### 4. 청구/정산 관리

**현재 상태**
- 전혀 구현 안 됨

**필요한 것**
- 고객별 계약 금액 관리
- 월별 청구 내역 자동 생성
- 미수금 추적

**임팩트**
- 영세 업체 사장이 "이번 달 얼마 받아야 하는지" 수기로 관리 중
- 회계 투명성 필요한 아파트 고객에서 특히 중요

---

### 5. 보건소 신고 연동

**현재 상태**
- 증명서는 생성되나 보건소 신고는 수작업

**필요한 것**
- 최소: 신고서 양식 자동 생성 (데이터 연동)
- 이상: 보건소 시스템 API 연동 (가능 여부 조사 필요)

**임팩트**
- 법적 의무 이행 자동화
- 경쟁사(케이웍스, RLMS)가 집중하는 영역 → 따라잡기 필요

---

### 6. 통계/보고서

**현재 상태**
- 대시보드는 "오늘/이번 주/이번 달" 수준

**필요한 것**
- 월별 추이 그래프
- 시설 유형별 분석
- 직원별 실적
- 연간 비교

**임팩트**
- 관리자 의사결정 지원
- 고객(입대위 등)에게 실적 리포트 제공 가능

---

## P2 — 사용성/확장

### 7. 일괄 방문 등록

**현재 상태**
- 방문 1건씩 수동 등록 (모달)

**필요한 것**
- 정기소독 고객 30곳에 대해 이번 달 일정 일괄 생성
- 계약 기반 자동 생성으로 대체 가능 (P0 #3이 해결되면 자연스럽게 해소)

---

### 8. 고객 포털

**현재 상태**
- 시설 관리자(아파트 관리소장 등)가 소독증명서를 직접 조회 불가
- 발급한 증명서를 매번 카톡/이메일로 전달해야 함

**필요한 것**
- 고객용 로그인 또는 공유 링크
- 증명서 자동 조회/다운로드

**임팩트**
- 커뮤니케이션 부담 경감
- 고객 락인(Lock-in) 효과

---

### 9. 알림/리마인더

**현재 상태**
- 예정 방문 전날 알림 없음
- 미완료 건 경고 알림 없음 (대시보드 표시만)

**필요한 것**
- 방문 전날 푸시/카톡 알림 (담당자)
- 미완료 건 경고 알림 (관리자)
- 계약 만료 알림

---

### 10. 루트 최적화

**현재 상태**
- 구현 안 됨

**필요한 것**
- 하루 방문 순서 최적화 제안

**임팩트**
- 한국 시장에서 우선순위 낮음 (하루 3~8건 수준)
- 미국 GorillaDesk 대비 한국 필요성 낮음

---

## 핵심 요약

```
현재 방역매니저가 해결하는 문제:
  "방문 기록 + 소독증명서 발급"
  → 케이웍스/RLMS 수준

방역매니저가 해결해야 하는 문제:
  "올해 이 고객에게 몇 번 더 가야 하지?"     ← 없으면 엑셀로 회귀
  "이번 달에 일정 안 잡은 고객이 있나?"      ← 핵심 가치
  "연간 법정 의무를 제대로 지키고 있나?"     ← 과태료 방지
```

**가장 임팩트가 큰 흐름**:
```
정기 계약 등록
  → 시설 유형 기반 연간 스케줄 자동 생성
    → 월별 "미배정 고객" 자동 추천
      → 방문 완료 → 증명서 자동 발급
        → 의무소독 현황 추적
          → 계약 만료 알림 → 갱신
```

이 흐름이 구현되면 "증명서 발급 도구"에서 **"방역 업무 전체를 관리하는 SaaS"**로 레벨업.

## 우선순위 로드맵 제안

| 단계 | 기능 | 근거 |
|------|------|------|
| **Phase 1** | P0 #3 정기계약 관리 | 모든 기능의 기반 데이터 |
| **Phase 2** | P0 #2 이번 달 방문 자동 추천 | 계약 기반 자연스럽게 도출 |
| **Phase 3** | P0 #1 연간 의무소독 현황 추적 | 계약 + 방문 데이터 결합 |
| **Phase 4** | P1 #4 청구/정산 | 계약에 금액 데이터 있으므로 확장 용이 |
| **Phase 5** | P1 #6 통계/보고서 | 축적된 데이터 활용 |
| **Phase 6** | P2 #8 고객 포털 | 차별화 피처 |
| **Phase 7** | P1 #5 보건소 신고 연동 | 외부 시스템 의존, 리스크 있음 |
| **후순위** | P2 #9 알림, #10 루트 최적화 | 부가 기능 |
