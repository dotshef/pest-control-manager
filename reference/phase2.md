# Phase 2: 고객 관리 (Client Management)

## 목표
고객(시설) 등록/수정/조회 기능 구현. 시설 유형 선택 시 법정 소독 주기 자동 세팅.

## 선행 조건
- Phase 1 완료 (DB, 인증, 레이아웃)

## 작업 목록

### 2-1. 고객 등록
- [ ] 고객 등록 폼 — 시설명, 주소, 시설 유형(드롭다운), 면적, 담당자명, 연락처, 메모
- [ ] 시설 유형 선택 시 `facility_types` 마스터에서 주기 자동 표시
- [ ] 고객 등록 시 `schedules` 자동 생성 (현재 월 기준 하절기/동절기 판별 → 주기 설정)
- [ ] 첫 방문 예정일 입력 → 첫 visit 자동 생성

### 2-2. 고객 목록
- [ ] 고객 목록 페이지 — 테이블 형태 (시설명, 시설유형, 주소, 담당자, 연락처, 다음 방문일)
- [ ] 검색 기능 (시설명, 담당자명)
- [ ] 필터 — 시설 유형별, 담당 기사별, 활성/비활성
- [ ] 페이지네이션 또는 무한 스크롤

### 2-3. 고객 상세
- [ ] 좌측: 시설 정보 카드 (시설명, 유형, 주소, 면적, 담당자, 연락처, 메모)
- [ ] 우측 상단: 다음 방문 예정 카드 (날짜, 요일, "방문 예정" 상태 배지)
- [ ] 우측 하단: 요약 통계 3개 — 총 방문 횟수 / 완료율 / 증명서 발급 수
- [ ] 고객 정보 수정 기능
- [ ] 고객 비활성화 (soft delete)
- [ ] 하단: 방문 이력 테이블 (날짜, 소독방법, 사용약제, 담당기사, 증명서번호, PDF 다운로드)

### 2-4. 권한 처리
- [ ] admin만 고객 등록/수정 가능
- [ ] member는 본인 담당 고객만 조회

## 산출물
- 고객 CRUD 동작
- 시설 유형 → 소독 주기 자동 매핑
- 고객 등록 시 스케줄 자동 생성
