## 1. 제품 개요

### 1.1 비전

소독 방역업체가 엑셀과 종이 증명서에서 벗어나, 스케줄 관리·증명서 발급·고객 이력을 하나의 플랫폼에서 처리할 수 있는 SaaS.

### 1.2 문제 정의

- 수십~수백 고객의 법정 소독 주기를 엑셀로 관리하면 빠뜨리는 건이 발생한다.
- 빠뜨리면 고객이 100만 원 이상의 과태료를 받고, 업체는 계약을 잃는다.
- 매 방문마다 소독증명서를 수기 또는 한글/워드로 작성하는 데 하루 1~2시간을 소모한다.
- 고객별 방문 이력, 사용 약제 히스토리를 확인하려면 엑셀을 뒤져야 한다.

### 1.3 타깃 사용자

- **관리자 (admin)**: 소독 방역업체 대표 또는 사무 담당자. 스케줄 관리, 고객 등록, 증명서 발급 이력 확인.
- **현장 기사 (technician)**: 현장에서 소독 완료 체크, 증명서 생성 및 발송.

### 1.4 XYZ 가설

적어도 5%의 소독 방역업체는 스케줄 관리, 기사 관리, 소독증명서 발급에 페인을 느끼고 있을 것이다.

### 1.5 시장 규모

- 국내 방역소독업체: 약 8,000~11,000개
- 시장 규모: 약 1조 7천억 원 (2021년 통계청)
- 75% 이상이 연 매출 5억 미만의 영세업체
- 5% 타깃 시 약 400개 업체 확보 목표

## 3. MVP 범위

### 3.1 1순위 — 스케줄 관리 + 자동 리마인더 (1-A)

**왜 필요한가**: 편의가 아니라 매출 방어. 법정 주기를 빠뜨리면 고객이 과태료를 맞고 업체는 계약을 잃는다.

**기능 목록**:

- 고객 등록 시 시설 유형 선택 → 법정 소독 주기 자동 세팅
- 캘린더 뷰에서 이번 달/이번 주 방문 예정 건 확인
- 방문 하루 전, 방문 당일 담당자에게 알림 (MVP에서는 인앱 알림, 이후 알림톡 확장)
- 미완료 건 빨간색 표시 (missed 상태)
- 방문 완료 체크 시 다음 방문 예정일 자동 갱신

### 3.2 1순위 — 소독증명서 자동 생성 (1-B)

**왜 필요한가**: 매 방문마다 고객에게 증명서를 발급해야 한다. 하루 5건이면 수기 작성에만 1~2시간.

**기능 목록**:

- 소독 완료 체크 시 자동 입력 항목: 업체명, 소독일시, 소독 장소, 사용 약제명, 소독 방법
- "증명서 생성" 버튼 → 업체 로고 + 법정 양식 기반 소독증명서 PDF 즉시 생성
- 전송 버튼 → 고객의 카톡/문자로 즉시 발송 (MVP에서는 PDF 다운로드 + 링크 공유, 이후 알림톡 확장)
- 발급 이력 자동 저장
- 증명서 번호 자동 채번 (CERT-YYYYMMDD-00001)

### 3.3 1순위 — 고객 이력 관리 (1-C)

**왜 필요한가**: 반복 방문 사업이므로 고객별 히스토리 추적이 필수.

**기능 목록**:

- 고객 카드: 시설 정보, 담당자, 연락처, 메모
- 방문 이력 타임라인: 날짜별 방문 기록, 사용 약제, 소독 방법
- 고객 검색 및 필터: 시설 유형별, 담당 기사별, 방문 상태별

| 식품접객업소 (연면적 300㎡ 이상) | 1개월 | 2개월 |

### 4.2 사용자 권한

| 역할 | 고객 등록/수정 | 스케줄 관리 | 방문 완료 체크 | 증명서 생성 | 기사 관리 |
| --- | --- | --- | --- | --- | --- |
| admin | O | O | O | O | O |
| technician | X | 본인 건만 조회 | O | O | X |

### 4.3 핵심 사용자 플로우

**플로우 A — 고객 등록 → 스케줄 자동 생성**:

1. 관리자가 고객 정보 입력 (시설명, 주소, 담당자 등)
2. 시설 유형 선택 (드롭다운)
3. 현재 월 기준으로 하절기/동절기 판별 → 소독 주기 자동 세팅
4. 첫 방문 예정일 입력
5. 스케줄 생성 → 캘린더에 반영
6. 주기에 따라 이후 방문 건(visits)이 자동 생성

**플로우 B — 현장 소독 완료 → 증명서 발급**:

1. 기사가 캘린더 또는 오늘 할 일 목록에서 방문 건 선택
2. 소독 방법, 사용 약제 입력 (프리셋에서 선택 또는 직접 입력)
3. "소독 완료" 버튼 → status가 completed로 변경, completed_at 기록
4. "증명서 생성" 버튼 → PDF 즉시 생성
5. PDF 다운로드 또는 링크 공유
6. 다음 방문 예정일이 schedule에 따라 자동 갱신

**플로우 C — 미완료 건 확인**:

1. 예정일이 지난 방문 건은 자동으로 status: missed로 변경 (cron 또는 앱 로딩 시 체크)
2. 캘린더에서 빨간색으로 표시
3. 관리자가 missed 건을 확인 → 재방문 스케줄 수동 등록 또는 기존 건 완료 처리

## 9. 성공 지표

### 9.1 가설 검증 단계 (랜딩페이지)

- 사전등록 전환율 5% 이상
- WTP(지불 의향 가격) 응답에서 월 3만원 이상 비율 60% 이상

### 9.2 MVP 단계

- 첫 달 유료 전환 업체 10곳 이상
- 주간 활성 사용률 (WAU/가입) 60% 이상
- 증명서 발급 건수 업체당 월 20건 이상
- NPS 40 이상

### 9.3 성장 단계

- 월 MRR 3,000만원 도달 시점: 출시 후 12개월 이내
- 베이직 → 플러스 업그레이드 전환율 20% 이상
- 월간 이탈률 (churn) 5% 이하
